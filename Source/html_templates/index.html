DeepSec
DEciding Equivalence Properties in SECcurity protocols

DeepSec is a fully automatic verification tool that can check equivalences between processes.

Features : DeepSec accepts as inputs processes in an applied-pi calculus like model. It can accept any destructor subterm convergent rewriting system. Finally, DeepSec can verify trace equivalence in the classical semantics, meaning that internal communication are allowed on both public and private channels.

In upcoming releases, we plan to implement the following features:
  • Allowing any destructor convergent rewriting system, not only subterm (although termination will be only guaranteed for subterm convergent rewriting systems).
    • classical one.
  • Adding the eavesdrop semantics where internal communication are only allowed on private channels but the attacker can eavesdrop on public channels.
  • Verifying observational equivalence : We will ensure partial correctness for any destructor convergent rewriting system. Moreover, termination will be guaranteed for destructor subterm convergent rewriting system.
  • A complete documentation for both users (for the syntax of input files accepted by DeepSec) and developpers (for a detailed description of the ml interfaces of the source code).
