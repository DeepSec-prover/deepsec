DeepSec
DEciding EQuivalence Properties in SECcurity protocols
<

DeepSec is a fully automatic verification tool that can check equivalences between processes.

Features : DeepSec accepts as inputs processes in an applied-pi calculus like model. It can accept any destructor subterm convergent rewriting system. Finally, DeepSec can verify trace equivalence in the classical semantics, meaning that internal communication are allowed on both public and private channels.

In upcoming releases, we plan to implement the following features:
  • Allowing any destructor convergent rewriting system, not only subterm (although termination will be only guaranteed for subterm convergent rewriting systems).
  • Adding the private semantics where internal communication are only allowed on private channels. Verification should be much faster with this semantics. Notice that this semantics is not equivalent to the classical one.
  • Adding the eavesdrip semantics where internal communication are only allowed on private channels but the attacker can eavesdrop on public channels.
  • Verifying observational equivalence : We will ensure partial correctness for any destructor convergent rewriting system. Moreover, termination will be guaranteed for destructor subterm convergent rewriting system.
  • Implementing DeepSec for destributed computing.
  • A complete documentation for both users (for the syntax of input files accepted by DeepSec) and developpers (for a detailed description of the ml interfaces of the source code).

Testing interface

A testing interface is available for developer to check any update they provide to the code. It can also be used by users to verify more in depth that DeepSec is behaving as expacted. When the testing interface is enabled, every input file given to deepsec will generate new tests that needs to be validated. Once a test a test is validated, it will become part of our regression test suite.

An HTML display of the validated tests, tests to check and tests of the regression suite that failed the verification process is available here.

To enable this testing interface, DeepSec must be recompile as follows.

make testing make

Once compile, a new executable is created test_deepsed that has two main functionalities:

  • It can verify all the validated tests from the regression test suite
  • It can validate new tests and incorporate them to the regression test suite

WARNING : Using the testing interface will slow down heavely the verification of of processes as it generates multiple tests, display them on html pages, etc. To disable the testing interface, one should once again compile DeepSec as follows.

make without_testing make
