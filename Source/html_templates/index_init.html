Testing interface

A testing interface is available for developer to check any update they provide to the code. It can also be used by users to verify more in depth that DeepSec is behaving as expacted. When the testing interface is enabled, every input file given to deepsec will generate new tests that needs to be validated. Once a test a test is validated, it will become part of our regression test suite.

An HTML display of the validated tests, tests to check and tests of the regression suite that failed the verification process is available here.

To enable this testing interface, DeepSec must be recompile as follows.

make testing make

Once compile, a new executable is created test_deepsed that has two main functionalities:

  • It can verify all the validated tests from the regression test suite
  • It can validate new tests and incorporate them to the regression test suite

WARNING : Using the testing interface will slow down heavely the verification of of processes as it generates multiple tests, display them on html pages, etc. To disable the testing interface, one should once again compile DeepSec as follows.

make without_testing make
